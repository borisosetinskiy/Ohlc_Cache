# OHLC Cache

Высокопроизводительная система кэширования OHLC (Open-High-Low-Close) данных для финансовых инструментов, построенная на Spring Boot с использованием Chronicle Map для хранения данных.

## 🎯 Описание

OHLC Cache - это специализированная система кэширования, предназначенная для работы с большими объемами финансовых данных в реальном времени. Система обеспечивает:

- **Высокую производительность** благодаря использованию off-heap памяти
- **Эффективное хранение** временных рядов OHLC данных
- **Быстрый поиск** по временным диапазонам
- **Масштабируемость** для работы с миллионами свечей

Основные настройки в `src/main/resources/application.yaml`:

```yaml
spring:
  application:
    name: Ohlc_Cache

cache:
  disable: false
  chronicle:
    time-series:
      base-path: "data/timeseries-chronicle"
  index:
    name: chronicle
    time-series:
      queue-size: 131072
      base-path: "data/timeseries-index"
  cleanup:
    enabled: true
    retention-period-ms: 604800000  # 7 дней в миллисекундах
    schedule-interval-ms: 3600000   # 1 час в миллисекундах
```

## 📊 Модели данных

### Candle (Свеча)

Основная структура данных для OHLC информации:

```java
public interface Candle {
    double getHigh();      // Максимальная цена
    double getLow();       // Минимальная цена
    double getOpen();      // Цена открытия
    double getClose();     // Цена закрытия
    double getVolume();    // Объем торгов
    long getTime();        // Временная метка
    int getTick();         // Номер тика
}
```

**Параметры:**
- **Длительность (мс)**: длительность одной свечи в миллисекундах
- **Свечей в чанке**: количество свечей, которые помещаются в один чанк (4320 для большинства периодов, 1220 для месяца, 100 для года)
- **Макс. чанков**: максимальное количество чанков в кэше для данного периода
- **Индекс (сек)**: количество секунд, которое помещается в индекс (TimeUnit.DAYS.toSeconds(N))

## 🔧 API

### Вставка данных

#### Одиночная вставка свечи:
```java
candleTimeSeriesCache.insert(
    "AAPL",           // Символ
    2,                // Period.durationId (M5 = 2)
    1640995200000L,   // Временная метка
    150.0,            // Open
    152.0,            // High
    149.0,            // Low
    151.0,            // Close
    1000000.0,        // Volume
    1                 // Tick
);
```

#### Пакетная вставка:
```java
Candle[] candles = new Candle[100];
// ... заполнение массива свечей
candleTimeSeriesCache.batchInsert("AAPL", 2, candles, 0, 100);  // Period.durationId (M5 = 2)
```

### Чтение данных

#### Поиск по временному диапазону:
```java
Candle[] result = new Candle[1000];
int count = candleTimeSeriesCache.batchRead(
    result,                    // Массив для результатов
    "AAPL",                    // Символ
    MarketType.STOCK,          // Тип рынка
    2,                         // Period.durationId (M5 = 2)
    1640995200000L,            // Начальная временная метка
    0,                         // Смещение в массиве результатов
    1000,                      // Количество свечей
    Direction.FORWARD,         // Направление поиска
    SearchMode.EXACT           // Режим поиска
);
```

## 🎯 Особенности производительности

### Off-heap память
- Использование Chronicle Map для хранения данных вне кучи
- Снижение нагрузки на GC
- Увеличение доступной памяти для данных

### Оптимизированные структуры
- Flyweight паттерн для объектов Candle
- Бинарный поиск в чанках данных
- Эффективное управление памятью

### Асинхронная индексация
- Отдельный поток для обновления индексов
- Неблокирующие операции записи
- Высокая пропускная способность

## 🧹 Управление жизненным циклом данных

### Автоматическая очистка (TTL)

Система поддерживает автоматическое удаление устаревших данных на основе TTL (Time To Live):

#### Настройки очистки
```yaml
cache:
  cleanup:
    enabled: true                    # Включение автоматической очистки
    retention-period-ms: 604800000   # Период хранения (7 дней)
    schedule-interval-ms: 3600000    # Интервал запуска (1 час)
```

#### Технические метаданные
Каждый чанк содержит технические метаданные:
- `createdTimestamp` - время создания чанка
- `lastAccessTimestamp` - время последнего доступа
- Автоматическое обновление при операциях чтения/записи

### Ручное управление

#### Удаление конкретного чанка:
```java
boolean deleted = candleTimeSeriesCache.deleteChunk(
    "AAPL",           // Символ
    2,                // Period.durationId (M5 = 2)
    1640995200000L    // Временная метка начала чанка
);
```

#### Ручная очистка устаревших данных:
```java
int deletedCount = candleTimeSeriesCache.cleanupExpiredChunks(
    7 * 24 * 60 * 60 * 1000L  // 7 дней в миллисекундах
);
```

### REST API для управления

#### Получение статистики:
```bash
GET /api/cache/statistics
```

#### Информация о настройках очистки:
```bash
GET /api/cache/cleanup/info
```

#### Ручной запуск очистки:
```bash
POST /api/cache/cleanup/manual?retentionPeriodMs=604800000
```

#### Удаление конкретного чанка:
```bash
DELETE /api/cache/chunks/AAPL?duration=2&chunkStartTimestamp=1640995200000
```

#### Обновление периода хранения:
```bash
PUT /api/cache/cleanup/retention?retentionPeriodMs=86400000
```

#### Включение/выключение автоматической очистки:
```bash
PUT /api/cache/cleanup/enabled?enabled=true
```

## 📈 Мониторинг

### Логирование
Система использует SLF4J для логирования. Основные события:
- Инициализация кэшей

- Операции очистки и удаления

### Метрики производительности
- Время выполнения операций
- Размер кэша
- Количество чанков и индексов

## 🚨 Известные ограничения

1. **Размер чанка:** Ограничен количеством свечей в чанке
2. **Временные метки:** Должны быть в миллисекундах
3. **Символы:** Максимальная длина 24 символа
4. **Память:** Требует значительного объема RAM
